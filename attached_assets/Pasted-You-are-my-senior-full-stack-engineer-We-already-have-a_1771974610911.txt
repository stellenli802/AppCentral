(1) PERMISSIONS SYSTEM (platform feel)
========================================

Goal: When a mini-app requests sensitive APIs (location/auth), the host enforces permissions with a native-looking consent modal, stores per-mini-app permissions, and exposes a “Permissions” settings page.

Requirements:
- Add per-mini-app permissions state:
  - Store in DB if available; otherwise local storage (persist across reload).
  - Keyed by {userId, miniAppId, permissionName}.
  - Support: "location", "auth" (auth = REQUEST_AUTH_CODE).
- In the runner bridge handler:
  - On REQUEST_LOCATION:
    - If permission already granted → respond with LOCATION immediately.
    - If not granted → show modal:
      “{MiniAppName} wants to access your Location”
      Buttons: [Allow once] [Always allow] [Deny]

        - Always allow: persist.
        - Deny: respond with an ERROR message type.
  - On REQUEST_AUTH_CODE:
    - Same flow, but permission name = "auth".
- Add new bridge response type:
  - HOST_ERROR { requestType, code, message }
- Update mini-apps to gracefully handle HOST_ERROR and display a friendly message in-app.

Permissions Settings Page:
- Route: /settings/permissions
- UI: list each mini-app with toggles:
  - Location: On/Off
  - Auth: On/Off
- Toggling Off should revoke persistent permission.
